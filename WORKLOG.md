### Key Decisions
- **Budget-only PATCH semantics di PUT /trips/:id** â€” field `budget` yang omitted = "leave unchanged", `budget: 0` = "explicitly clear". Field lain tetap full-replacement. Ini exception yang sengaja didokumentasikan di comment `UpdateInput.Budget` karena hanya `budget` yang punya masalah `*float64 â†’ 0` coercion; field lain tidak terdampak.

- **Mock `Update` diperbaiki, bukan di-workaround** â€” mock lama return `m.updateResult` (nil) dan membuang input, sehingga test tidak bisa meng-assert field assignment di usecase. ### API Contract Nuance
`PUT /trips/:id` sekarang punya semantics asymmetric untuk `budget`:

- `budget: 0` â†’ budget trip di-set ke 0 (explicit clear)

Field lain (`title`, `description`, `start_date`, `end_date`, `base_currency`, `cover_image_url`, `notes`) tetap full-replacement (kosong/null di-overwrite). Jika nanti ingin PATCH semantics untuk semua field optional, itu refactor terpisah yang lebih besar.

### Fix 1 — Mutation Invalidation (create/delete/update not appearing)

**Root cause**: `refetchOnMount: false` in QueryClient defaults meant stale queries from `invalidateQueries` never refetched when navigating back. Also most `invalidateQueries` calls were missing `refetchType: "all"`.

- **`providers.tsx`**: Changed `refetchOnMount` from `false` to `true`. Stale queries now refetch on navigation. With `staleTime: 5min`, normal navigation still doesn't cause unnecessary refetches.
- **All mutation hooks**: Added `refetchType: "all"` to every `invalidateQueries` call across 6 files:
  - `useTrips.ts` — create, update, delete, createFromDraft, updateDayNotes
  - `useExpenses.ts` — invalidateAll() helper
  - `useActivities.ts` — create, update, delete, reorder
  - `useAccommodations.ts` — create (update/delete already had it)
  - `useTransportations.ts` — create (update/delete already had it)
- **Memory**: Added pattern #9 documenting `refetchType: "all"` requirement

### Fix 2 — 429 on First Generate Attempt (Rate Limiter Overhaul)

**Root cause**: Rate limiter ran as global middleware BEFORE auth middleware, so `UserIDKey` was never set. All requests used IP-based keys. On localhost, multiple users/preflights shared the same bucket, causing 429 on first attempt.

**Backend changes:**
- **`pkg/jwt/jwt.go`**: Added `ExtractUserIDUnverified()` — parses JWT claims without signature validation, for rate-limit bucketing only.
- **`middleware/rate_limiter.go`**: RateLimiter now accepts JWT service. Identity priority: (1) context `UserIDKey`, (2) JWT header extraction, (3) IP fallback. Fixed summary path matching — colon `:id` syntax replaced with `HasPrefix + HasSuffix`.
- **`cmd/server/main.go`**: Rate limiter moved from global to API group. Constructor passes `jwtSvc`.
- **`autogen/handler.go`**: Cooldown check changed from TTL-check-then-Set (race-prone) to atomic `SETNX`. Cooldown reserved BEFORE LLM call.
- **`summary/handler.go`**: Added same Redis SETNX cooldown pattern (`WithSummaryRateLimit`). Previously per-trip (DB `UpdatedAt`), now per-user like generate trip.

| Lapis | Scope | Key Redis |
|-------|-------|-----------|
| Sliding window 5/min | Per-user | `ratelimit:llm:{userID}` |
| Generate cooldown 300s | Per-user | `ratelimit:autogen:cooldown:{userID}` |
| Summary cooldown 300s | Per-user | `ratelimit:summary:cooldown:{userID}` |
